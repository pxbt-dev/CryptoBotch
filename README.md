# CryptoBotch - a botched version of Cryptowatch

A live cryptocurrency trading dashboard built on Binance WebSocket streams. Supports any Binance trading pair with real-time charts, order book, and recent trades. Inspired by the late CryptoWatch (RIP).

## Stack

**Backend**
- Java 21, Spring Boot 3.4.1
- Spring WebSocket (STOMP) — live data to the browser
- Spring WebFlux (WebClient) — reactive Binance REST calls
- ta4j — technical indicator calculations

**Frontend**
- React 19, Vite 8
- lightweight-charts v5 — candlestick / Heikin Ashi / line / bar charts
- @stomp/stompjs — shared WebSocket connection
- Tailwind CSS v4

**Data source**: Binance WebSocket Streams + REST API

## Features

- Live price charts with multiple timeframes (1m → 1W)
- Chart types: Candles, Heikin Ashi, Line, Bars — with custom up/down colours
- Technical indicators: EMA, SMA, Bollinger Bands (more coming)
- Live order book and recent trades
- Multi-chart grid layout (up to 9 simultaneous charts)
- Search and add any Binance trading pair (BTCUSDT, BTCEUR, ETHUSDT, etc.) with autocomplete
- Resizable order book / recent trades panel

## Running locally

**Backend**
```bash
mvn spring-boot:run
```
Starts on port 8080.

**Frontend** (dev server with hot reload)
```bash
cd ui && npm install && npm run dev
```

**Frontend** (build into jar)
```bash
cd ui && npm run build
```
Outputs directly to `src/main/resources/static/` so the Spring Boot jar serves the UI.

## Deployment (Railway)

The included `Dockerfile` handles the full build in three stages:

1. Node 22 — installs dependencies and builds the React app
2. Maven / Java 21 — copies the built frontend into `src/main/resources/static/` then packages the jar
3. Eclipse Temurin 21 JRE — runs the jar

Railway injects a `PORT` environment variable which the app respects via `${PORT:8080}` in `application.yml`.

## Environment variables

| Variable | Default | Description |
|----------|---------|-------------|
| `PORT`   | `8080`  | HTTP port (set automatically by Railway) |
